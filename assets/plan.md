Introduction au MCP (15 min)Théorie (10 min) : Qu'est-ce que le MCP et son Architecture ?L'Analogie parfaite : MCP est à l'IA ce que le port USB-C est à l'électronique. C'est un standard universel pour connecter des modèles d'IA (Claude, ChatGPT) à des sources de données et des outils externes, peu importe le langage ou l'environnement.Les Participants (L'Architecture) :MCP Host : L'application IA globale (ex: Claude Desktop, VS Code, Cursor).MCP Client : Le composant interne du Host qui gère une connexion unique vers un serveur précis.MCP Server : Le programme (local ou distant) qui expose le contexte ou les outils.Les Deux Couches du protocole :Data Layer : Basée sur JSON-RPC 2.0. Gère le cycle de vie (négociation des capacités) et les requêtes/réponses.Transport Layer : Standard I/O (Stdio) pour les serveurs locaux exécutés sur la même machine, ou Streamable HTTP / Server-Sent Events (SSE) pour les serveurs distants.Schéma 1 : L'Architecture Host - Client - ServeurExtrait de codegraph LR

Le Client MCP en Action (35 min)Théorie (15 min) : Le cycle de vie et les Primitives du ClientLa Négociation (initialize) : Lors de la connexion, le client et le serveur s'échangent leurs capacités respectives (ex: "Je supporte les outils et les notifications").Le Contrôle Utilisateur (Human-in-the-loop) : Le client est garant de la sécurité. Les actions sensibles des serveurs nécessitent l'approbation explicite de l'utilisateur.Les Primitives du Client (Ce que le client offre au serveur) : Le client n'est pas passif, il expose des fonctionnalités pour enrichir le serveur.Fonctionnalité ClientDescriptionCas d'usageElicitationPermet au serveur de suspendre une action pour demander des informations structurées à l'utilisateur.Demander les préférences de siège lors d'une réservation de vol.RootsIndique au serveur les limites du système de fichiers sur lequel il est autorisé à opérer.Restreindre l'accès à un dossier projet spécifique.SamplingPermet au serveur de demander au modèle IA du client de générer des complétions autonomes.Analyser une liste de 47 vols sans intégrer de SDK IA côté serveur.Pratique (20 min) : Connecter et exploiter un MCP distant (GitHub)Objectif : Utiliser Claude Code ou VS Code pour interagir avec GitHub via MCP.Étapes synthétiques pour les participants :Configuration du Client : Ajouter la configuration d'exécution du serveur MCP GitHub dans le fichier du client (claude_desktop_config.json ou config VS Code).Authentification : Fournir le Token d'Accès Personnel (PAT) GitHub.Exploitation : Demander à l'IA : "Utilise tes outils GitHub pour accéder au repo https://github.com/t0sa/club-genai-mcp et récupère le fichier agent.md dans /agent."Observation : Analyser comment l'application demande l'approbation à l'utilisateur avant d'exécuter la requête réseau (CallToolRequest).3. Construire son Serveur MCP (50 min)Théorie (15 min) : Les Primitives du Serveur et l'ÉcosystèmeChaque primitive est contrôlée par un acteur différent pour garantir une intégration fluide.Primitive ServeurDescriptionContrôlé parExempleOutils (Tools)Fonctions actives exécutées à la demande.Le Modèle (IA)searchFlights(), createTask()RessourcesDonnées passives pour alimenter le contexte.L'ApplicationAccès au contenu d'un calendrierPromptsTemplates d'instructions réutilisables.L'UtilisateurSlash command /plan-vacationSupport Multi-langages : Il existe des SDK officiels de différents "Tiers" pour TypeScript, Python, C#, Go, Java, Rust, Ruby, etc.Alerte Technique (Le piège du Stdio) : Pour les serveurs utilisant le transport local Stdio, il est strictement interdit d'utiliser print() ou console.log() classique. Écrire sur la sortie standard (stdout) corrompt les messages JSON-RPC. Il faut impérativement utiliser stderr (ex: console.error(), sys.stderr).Schéma 2 : Interaction des Primitives (Exemple de Workflow)Extrait de codegraph TD

Pratique (35 min) : Création d'un Serveur MCP "Task Tracker"Objectif : Créer un serveur MCP (en Python ou TypeScript via le SDK) connecté à une API OpenAPI.Étapes synthétiques pour les participants :Initialisation : Utiliser l'agent récupéré (agent.md) pour scripter la structure de base (via uv pour Python ou npm pour Node).Développement : L'API cible tourne sur http://localhost:8080. Dire à l'agent de générer les outils MCP suivants à partir de l'OpenAPI :get_all_tasks()get_latest_task()create_task(pseudo, nom_de_tache, avancement)Bonnes Pratiques : S'assurer que l'agent génère bien des logs sur stderr et implémente correctement les schémas via Pydantic ou Zod.Test & Debug : Lancer l'inspecteur web MCP (mcp dev ...) pour tester manuellement les outils avant de les soumettre à Claude.4. Conclusion et Ouverture (10 min)Synthèse (5 min)MCP permet de déporter la complexité d'intégration métier vers des serveurs réutilisables.La couche de transport agnostique permet d'utiliser MCP en local pur (sans latence réseau) ou à distance via HTTP.La sécurité reste au centre avec des concepts d'approbation et un modèle "Human-in-the-loop".Ouverture (5 min)L'écosystème MCP grandit vite avec l'ajout expérimental de fonctionnalités comme les "Tasks" (opérations longues) et les "Notifications" (mises à jour en temps réel de la liste des outils).Call to Action : Pour approfondir l'architecture, la sécurité avancée ou construire des connecteurs distants complexes, direction le cours gratuit d'Anthropic :Anthropic Skilljar - Model Context Protocol Advanced Topics
