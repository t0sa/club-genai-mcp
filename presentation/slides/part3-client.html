Cycle de Vie de la Connexion

Trois étapes clés pour qu'un client MCP soit opérationnel.

🤝

1. Handshake

initialize()

Le client lance le serveur et vérifie la version du protocole pour assurer la compatibilité.

→
📋

2. Listing

tools/list

Le client demande la liste des outils et ressources disponibles sur le serveur.

→
⚡

3. Call

tools/call

Le client envoie une requête d'exécution avec des arguments JSON typés.

{ "method": "tools/call", "params": { "name": "add_numbers", "arguments": { "a": 5, "b": 3 } } }

Gestion des Permissions

Le client est le garde-fou entre l'IA et les actions sensibles.

SANS VALIDATION

L'IA peut exécuter n'importe quel outil automatiquement — risque de dérive involontaire.

⚠️ Suppression de fichiers non désirée
⚠️ Envoi d'e-mails non approuvés

AVEC VALIDATION

✅L'utilisateur approuve chaque outil sensible
✅Possibilité de refuser l'exécution
✅Audit trail des actions

PATTERN DE VALIDATION

// Avant d'exécuter l'outil
if (tool.requiresApproval) {
  const ok = await askUser(
    `Autoriser "${tool.name}" ?`
  );
  if (!ok) return;
}
await client.callTool(tool.name, args);

"Le client doit marquer certains outils comme sensibles pour déclencher une confirmation obligatoire."
