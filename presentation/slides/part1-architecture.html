Qu'est-ce que MCP ?

DÉFINITION

Un protocole standardisé qui permet à une IA de communiquer avec des sources de données et des outils externes.

ANALOGIE

C'est comme USB pour les outils d'IA : un connecteur universel qui remplace des intégrations ad-hoc pour chaque service.

🔌

Un protocole

Open-source, développé par Anthropic

Remplace les intégrations custom par un standard unique

Architecture Hub-and-Spoke

L'hôte est le hub central qui communique avec N serveurs via le même protocole.

🤖
Hôte (Hub)
Claude / IDE
MCP
Serveur A — Fichiers
Serveur B — Base de données
Serveur C — API Externe

Chaque serveur est léger et indépendant — un seul protocole pour tous les connecter.

Les 3 Piliers de MCP

🏠 L'Hôte (Host)

L'application IA qui veut utiliser des données.

Claude Desktop VS Code IDE
🔗 Le Client

Le composant au sein de l'hôte qui établit et gère la connexion.

Responsable du handshake et de l'orchestration.

⚙️ Le Serveur

Le programme léger qui expose les capacités à l'IA.

  • 📄 Resources — données en lecture
  • 🛠️ Tools — actions exécutables
  • 💬 Prompts — templates pré-configurés

"Le serveur donne des super-pouvoirs à l'IA sans qu'elle ait besoin de tout re-implémenter."

Le Mécanisme de Transport

Comment le client et le serveur se parlent concrètement.

📟
stdio Local
  • ✅ Flux stdin / stdout standard
  • ✅ Parfait pour les outils locaux
  • ✅ Simple à implémenter
  • ⚠️ Limité à la machine locale
🌐
SSE (HTTP) Distant
  • ✅ Server-Sent Events over HTTP
  • ✅ Serveurs distants accessibles
  • ✅ Scalable et déployable
  • ⚠️ Plus complexe à sécuriser

Tous les échanges utilisent JSON-RPC 2.0 comme format de message.
