Les 3 Règles d'Or

🔒
Isolation

Toujours faire tourner les serveurs dans des environnements contrôlés (Docker, sandbox, droits restreints). Un serveur compromis ne doit pas compromettre le système entier.

Docker Droits UNIX minimaux Network isolation
✅
Validation

Utiliser des schémas JSON (Zod en TS, Pydantic en Python) pour valider toutes les entrées. Ne jamais faire confiance à un paramètre non typé.

Zod (TypeScript) Pydantic (Python)
📝
Logs sur stderr uniquement

Le transport stdio utilise stdout pour le protocole JSON-RPC. Tout log sur stdout corrompt le flux et casse la communication. Toujours rediriger vers stderr.

console.error("debug log"); // ✅ stderr

Ce Qu'il Faut Retenir

🏗️ ARCHITECTURE

  • ✅ Hub-and-Spoke : 1 hôte ↔ N serveurs
  • ✅ 3 piliers : Hôte, Client, Serveur
  • ✅ Transport : stdio (local) ou SSE (HTTP)
  • ✅ Protocole : JSON-RPC 2.0

⚙️ SERVEUR

  • ✅ 3 capacités : Resources, Tools, Prompts
  • ✅ SDK TypeScript ou Python
  • ✅ Validation Zod obligatoire
  • ✅ Périmètre d'accès strictement défini

🔗 CLIENT

  • ✅ Lifecycle : Handshake → Listing → Call
  • ✅ Garde-fou pour les outils sensibles
  • ✅ Approbation utilisateur requise
